# Security Policy

## Supported Versions

We actively monitor and patch security vulnerabilities for the following versions:

| Version | Supported          |
| :---    | :---               |
| 1.1.x   | Yes                |
| 1.0.x   | Critical Fixes Only|

## Reporting a Vulnerability

We take the security of Woof very seriously. If you discover any security-related issues or vulnerabilities (such as session leaks or cache poisoning), please **DO NOT** open a public issue. Instead, report it privately to our security team.

Please send a detailed report to: https://www.uhero.co.jp/uhero/inquiry/

### What to include in your report:
- A clear description of the vulnerability and its potential impact.
- Step-by-step instructions or a minimal proof-of-concept script to reproduce the issue.
- Details of your environment (PHP version, OS, Woof configuration).

### Our Process:
1. **Acknowledgement**: We will acknowledge receipt of your vulnerability report as soon as possible on a best-effort basis.
2. **Investigation**: Our team will investigate the issue and determine a remediation plan.
3. **Resolution**: We will work on a patch and coordinate a security release. We ask for your patience to keep the issue confidential until the official fix is published.
